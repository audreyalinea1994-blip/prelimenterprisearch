EA Frameworks are structured approaches that provide guidelines, methods, and models for developing and managing enterprise architecture.

2 Zachman Framework

Created by John Zachman in 1987, this is a classification schema that organizes architecture artifacts using a 6x6 matrix.

Key Characteristics:
  • â€¢ Two-dimensional classification matrix
  • â€¢ Descriptive, not prescriptive
  • â€¢ Technology independent
  • â€¢ Focuses on WHAT to document
The 6 Questions (Columns):
What (Data) How (Function) Where (Network) Who (People) When (Time) Why (Motivation)
The 6 Perspectives (Rows):
1. Planner (Scope) - Business context
2. Owner (Business) - Business concepts
3. Designer (System) - System logic
4. Builder (Technology) - Technology physics
5. Subcontractor (Component) - Tool configs
6. User (Operations) - Actual instances

TOGAF is a process-based framework that provides a detailed method for developing enterprise architecture through the ADM (Architecture Development Method).
